- what is microdata?
- data model: scoped name/value pairs
  - itemscope defines an item ( http://www.whatwg.org/specs/web-apps/current-work/multipage/microdata.html#items )
  - itemprop names the property ( http://www.whatwg.org/specs/web-apps/current-work/multipage/microdata.html#names:-the-itemprop-attribute )
  - values are stored in whatever attribute or child text makes sense, based on the elements involved ( http://www.whatwg.org/specs/web-apps/current-work/multipage/microdata.html#values )
  - itemtype defines the vocabulary. Using a shared vocabulary helps to semantically connect your data with other people's data on other pages and other sites
  - itemid for globally unique identifiers (if applicable)
  - itemref if an item's properties are defined elsewhere on the page (i.e. not on child elements of the itemscope)
- "Hello World" of microdata: licensing metadata on Creative-Commons-licensed photograph
- accessing microdata from JavaScript
  - document.getItems()
  - document.getItems("http://www.example.com/animals#cat")
  - element.properties
  - element.properties.names
  - element.properties.values
- Case study: an about page with contact information

Marking Up People

The easiest way to integrate microdata into a personal website is on your “about” page. You do have an “about” page, don’t you? If not, you can follow along as I extend this sample “about” page with additional semantics.

Let’s look at the markup first:
